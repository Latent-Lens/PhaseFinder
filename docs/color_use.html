CSS inventory

Color-use audit

This report maps the requested PhaseFinder color variables to the UI elements that consume them. It records the authored CSS value, an equivalent eight-digit hex value for translucent colors, the number of separate selector targets, the target element types and names, and a concise description of each color's role.

Scope The production app styles loaded by index.html, plus the standalone help.html stylesheet.
Counting unit Each distinct selector, state, or pseudo-element branch counts once; repeated runtime instances do not.
Excluded Generated documents and test pages that independently redefine similarly named tokens with unrelated palettes.

Interpretation notes

  • The count is a CSS-usage count, not a rendered-node count. For example, the global button selector is one target even though it can match many buttons, while hover and active selectors are counted separately.
  • Runtime counts vary for metadata filter menus, sortable headers and arrows, editable header inputs, DJF fit headings, and other data-dependent elements.
  • The specialized --th_bg header selectors overlap the global th rule; their eight targets therefore should not be added together to estimate unique table cells.
  • .stats_modal_card represents five concrete modal cards, so --shadow_strong reaches six concrete cards through two variable selectors; --progress_card_border reaches seven concrete surfaces through three selectors.
  • .info_icon::after and .help_link:hover currently have no matching markup or JavaScript-created element; .callout.limit .callout_title has no matching Help callout.
  • Two same-value literals are worth consolidating: .remove_columns_panel duplicates --shadow_strong, and .reset_session_button duplicates --restart_strong.

Token definitions are in css/base.css; mirrored Help-page tokens and Help-only targets are in css/help.css.
