User-controlled variables for the nine-stage DJF pipeline

The recommended UI exposes a small set of biologically meaningful controls by default and places numerical solver settings behind an Expert/Developer disclosure. A flat list of every constant would make it too easy to change convergence behavior without understanding whether the edit affects event exclusion, the fitted model, or only a warning.

Current state: the manual pipeline UI passes only Stage 4's histogram binCount and shared range. Stages 0–3 and 5–8 currently receive empty option objects and therefore use their shipped defaults.

Stages 1, 2, 3, and 7 should have explicit enable/skip controls. Structural QC should remain mandatory. Stages 4, 6, and 8 are required to produce a result. Stage 5 is useful inspection, but it is currently decoupled from Stage 6 and should be labeled accordingly.

Global and preflight controls

  • Channel mapping: choose the required DNA-area channel and show automatically detected DNA-height, DNA-width, FSC-area, SSC-area, and Time channels. Advanced users should be able to override an incorrect automatic mapping.
  • Optional stages: explicit enable switches for Time QC, FSC/SSC cell gating, pulse-geometry singlet gating, and contamination extension. “Run all” must respect these switches.
  • Expected DNA ratio: store one shared target, default 2.0, and propagate it to Stage 5 targetRatio, Stages 6–7 ratioTarget, and Stage 8 expectedRatio.
  • Scope: settings should default to all selected samples. A per-sample override must be explicit, visibly badged, and display both its inherited and effective value.
  • Linear DNA scale: keep the fitting histogram linear. Do not offer a logarithmic DNA axis as a fitting option.

Fixed Stage 6 initializer/model heuristics

Candidate peaks must reach 3% of the maximum. Initial candidate ratios are searched from 1.45 to 2.55, with ratio-penalty coefficient 6 and a fallback G2 search around the expected position. FWHM conversion, peak-area windows, nonnegative amplitudes, and the tapered quadratic Bernstein S-phase bridge are model implementation details. Keep them fixed until there is validation data supporting user control.

Keep internal, derived, or read-only

  • Structural, time-QC, scatter, singlet, combined, and final mask arrays.
  • Stage 1's input mask, MAD scale factor 1.4826, numerical epsilons, and finite-value rules.
  • Stage 2 componentCount; the current biological gate deliberately forces two components.
  • Stage 4/5 derived histogramMin and binWidth.
  • Fitted mu1, mu2, sigmas/CVs, amplitudes, S-bridge coefficients, aggregate parameter, debris amplitude, and debris decay. Show and export them, but do not overwrite a fit result.
  • Stage 8 channelNames, pulseGeometryAvailable, and parameterCount.
  • General background settings until a background model is specified and implemented.

Cross-stage behavior and guardrails

Do not create independent copies of shared scientific settings. If Stage 6 is configured with custom ratio, CV, or weighting constraints and Stage 7 receives its own defaults, the extension will silently refit the base parameters under different assumptions.

  • Store one global expected ratio. Keep separate, clearly named values for Stage 5's peak-search window, Stage 6's unlocked fit bounds, and Stage 8's warning tolerance because those have different meanings.
  • Synchronize Stage 6 and Stage 7 ratio, CV, unlock, and residual-weighting settings.
  • Either state that Stage 5 is diagnostic-only or add an explicit option that passes selected peaks to Stage 6 as initialization anchors.
  • Warn when both pulse-geometry singlet gating and aggregate modeling are disabled.
  • Warn when a manual histogram range clips retained events or chosen binning is sparse.
  • Any setting edit invalidates that stage and all downstream state. The UI should show that invalidation before rerunning, never leave stale downstream plots or reports visible.
  • Label every setting as affecting event masks, fit/model selection, or diagnostics/warnings only.

Validation requirements

  • wrapLowFraction < wrapHighFraction, both within [0,1].
  • Time-QC target bin size and all iteration/bin counts must be positive integers where applicable.
  • ratioMin ≤ ratioTarget ≤ ratioMax, with ratio values greater than 1.
  • 0 < cvMin ≤ cvMax.
  • Scatter confidence must map to a finite, nonnegative Mahalanobis-squared threshold.
  • Component weights, correlations, and fractions must remain within [0,1].
  • 0 < debrisTauMinFraction ≤ debrisTauMaxFraction.
  • Detection and improvement thresholds must be finite and nonnegative.
  • Absolute and fractional peak prominence should be mutually exclusive in the UI.
  • Manual histogram maximum must be greater than minimum, with clipping disclosed before execution.

Presets, control wording, and live feedback

Recommended presets

  • Recommended: exactly the shipped defaults.
  • Preserve events: gentler gates, clearly labeled as retaining more possible contaminants.
  • Stringent gating: tighter QC/gates, clearly labeled as rejecting more biological events.
  • Saved lab/instrument profile: a named, versioned configuration.

Non-default numeric presets should be validated against known controls before shipping. “Last used” must not silently replace the Recommended preset. Provide Reset Stage and Reset All actions.

Suggested tooltip language

  • Time-QC sensitivity: “Exclude acquisition periods when any robust metric exceeds this many MAD-based standard deviations. Lower values reject more events.”
  • Gate coverage: “Approximate fraction of the fitted FSC/SSC population inside the ellipse. Higher values retain more events.”
  • Singlet gate width: “Allowed distance from the pulse-geometry ridge, measured in MADs. Lower values reject more potential doublets.”
  • Count-weighted residuals: “Weight residuals approximately by counting noise. This changes the fitting objective and fit-comparison values.”
  • Contamination Auto: “Test aggregate and debris candidates and retain them only when residual structure and model-selection criteria support the added terms.”

Live feedback should show retained event count and percentage after Stages 0–3, range clipping and counts per bin after Stage 4, candidate and selected peaks after Stage 5, convergence and fitted ratio after Stage 6, candidate comparisons after Stage 7, and warning changes after Stage 8.

Persistence, export, and provenance

Every saved session or exported report should include:

  • Every effective setting and its source: default, preset, metadata-derived, inherited, or user override.
  • Application/git version, configuration-schema version, and preset version.
  • Sample checksum or stable identity, channel mappings, detected PnR, and PnR overrides.
  • Enabled/skipped stages, skip reasons, retained counts, and shared histogram range.
  • Solver convergence status and effective numerical settings.
  • Stage 7 candidate comparisons, selected model, Stage 8 warnings, and manual interventions.

Export a rerunnable machine-readable configuration, preferably JSON, plus a human-readable CSV/report. Pipeline settings and results are currently runtime-oriented; editable settings should not ship without session serialization and a versioned configuration schema.
